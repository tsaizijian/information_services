## 一、背景與動機

### 1.1 專題選擇動機

身心障礙照護機構在日常運作中，需要詳細記錄每位個案的照護情況、健康狀況、生命徵象等重要資訊。然而，傳統的紙本記錄方式存在以下問題：

- **資料查詢不便**：過往紀錄散落在不同的紙本文件中，需要花費大量時間翻找
- **資訊整合困難**：個案的照護紀錄、生命徵象、健康資料分散管理，無法快速掌握整體狀況
- **統計分析耗時**：月報表、統計資料需要人工彙整，容易出錯且耗費人力
- **交接效率低落**：班與班之間的交接需要口頭說明或手寫交接本，重要資訊可能遺漏
- **家屬溝通不易**：家屬無法即時了解家人在機構的照護情況

### 1.2 與機構需求的關聯

經過與照護機構老師的多次訪談，我們發現機構迫切需要一套數位化的照護紀錄系統，以：

1. **提升照護品質**：透過系統化記錄，確保每位個案都能獲得完整的照護
2. **降低人力成本**：減少紙本作業和重複登打的時間，讓照顧者能專注於照護工作
3. **強化資訊透明**：讓管理者、照顧者、家屬都能適時掌握個案狀況
4. **符合法規要求**：完整的電子紀錄有助於機構評鑑與法規遵循

### 1.3 預期改善的問題

本系統預期改善以下具體問題：

- ✅ **紀錄效率提升 80%**：從紙本手寫改為電子輸入，大幅縮短記錄時間
- ✅ **資料查詢即時化**：任何授權人員都能在 5 秒內查詢到所需資訊
- ✅ **統計報表自動化**：系統自動產生月報表、統計圖表，無需人工計算
- ✅ **減少資訊遺漏**：系統提醒與檢核機制，避免重要資訊漏記
- ✅ **促進家屬參與**：家屬帳號可隨時查看，增進家屬信任與參與度

## 二、需求與目標

### 2.1 使用者訪談結果

我們與照護機構進行了深入訪談，整理出三類主要使用者的需求：

#### **管理員需求**
- 完整的系統管理權限（使用者管理、班級管理、權限控制）
- 統計報表與數據分析功能
- 個案資料的新增、編輯、刪除權限
- 系統活動監控與稽核功能

#### **照顧者需求**
- 快速記錄照護紀錄（用餐、如廁、行為觀察等）
- 生命徵象測量與記錄（血壓、心率、體溫、血氧）
- 重要事項釘選與提醒功能
- 班級成員資訊快速查詢
- 交接事項清楚呈現

#### **家屬需求**
- 查看家人的照護紀錄與健康狀況
- 了解機構活動與照護內容
- 緊急狀況即時通知
- 安心且透明的資訊管道

### 2.2 系統核心目標

基於上述需求，我們設定以下核心目標：

| 目標項目 | 具體說明 | 達成指標 |
|---------|---------|---------|
| **數位化記錄** | 將紙本照護紀錄全面數位化 | 100% 電子化記錄 |
| **即時查詢** | 隨時隨地查詢個案資訊 | 查詢響應 < 3 秒 |
| **資料安全** | 確保個案隱私與資料安全 | 符合 GDPR 標準 |
| **操作簡便** | 介面直覺易用，減少訓練成本 | 30 分鐘內上手 |
| **行動友善** | 支援手機、平板等行動裝置 | RWD 響應式設計 |

### 2.3 資訊技術應用重點

1. **雲端即時同步**：使用 Firebase 實現多人即時協作
2. **權限分級管理**：三層權限（管理員、照顧者、家屬）確保資料安全
3. **資料視覺化**：圖表呈現生命徵象趨勢，一目了然
4. **智慧搜尋與篩選**：快速找到需要的個案或紀錄
5. **匯出功能**：支援 PDF、Excel 格式匯出報表

## 三、實現方法

### 3.1 技術架構

#### **前端技術**
- **框架**：Nuxt 3 (Vue 3 + Server-Side Rendering)
- **UI 框架**：PrimeVue 4 + Tailwind CSS
- **狀態管理**：Vue Composition API + Composables
- **表單驗證**：Valibot
- **圖表視覺化**：Chart.js + vue-chartjs

#### **後端服務**
- **BaaS 平台**：Firebase (Google Cloud)
  - Firebase Authentication：使用者認證
  - Cloud Firestore：NoSQL 資料庫
  - Cloud Storage：檔案儲存
  - Security Rules：資料存取權限控制

#### **開發工具**
- **版本控制**：Git
- **套件管理**：pnpm
- **程式語言**：TypeScript

### 3.2 系統架構圖

```

│           使用者介面層 (Frontend)              │
│    Nuxt 3 + Vue 3 + PrimeVue + Tailwind     │
```

### 3.3 資料庫設計

#### **主要 Firestore 集合**

| 集合名稱 | 說明 | 主要欄位 |
|---------|------|---------|
| `users` | 使用者帳號 | uid, displayName, role, email |
| `classes` | 班級資料 | className, location, contactPhone |
| `clients` | 個案基本資料 | name, gender, birthDate, classId, basicInfo |
| `records` | 照護紀錄 | clientId, recordType, content, recordDate, isPinned |
| `monthlyVitalSigns` | 每月生命徵象 | clientId, year, month, weight, bloodPressure, pulse, bloodOxygen |
| `vitalSignRecords` | 詳細生命徵象 | clientId, measuredAt, systolic, diastolic, heartRate, temperature |
| `physicalRecords` | 身高體重歷史 | clientId, height, weight, bmi, measuredDate |

### 3.4 主要功能模組

#### **1. 身份驗證系統**
- Email/密碼登入
- 角色權限控制（admin, caregiver, family）
- 自動登出與 Session 管理
- 密碼重設功能

#### **2. 個案管理模組** (`/clients`)
- 個案列表檢視（支援搜尋、篩選）
- 新增/編輯個案資料
- 個案詳情頁面（基本資訊、照護紀錄、生命徵象趨勢）
- 個案照片上傳
- 緊急聯絡人管理

#### **3. 照護紀錄模組** (`/records`)
- 快速記錄照護事項
- 紀錄類型分類（日常照護、健康觀察、活動參與、飲食、行為、特殊事件）
- 釘選重要紀錄
- 多條件篩選（個案、類型、日期範圍）
- 照片附件上傳

#### **4. 生命徵象管理** (`/vital-signs`)
- 月度生命徵象記錄（體重、血壓、心率、血氧）
- 詳細生命徵象測量（血壓、心率、體溫、血氧、血糖）
- 趨勢圖表視覺化
- 異常值警示
- 匯出報表（PDF/Excel）

#### **5. 班級管理** (`/classes`) - 僅管理員
- 班級新增/編輯/刪除
- 班級成員管理
- 班級統計資訊
- 照顧者分配

#### **6. 儀表板** (`/dashboard`)
- 系統統計概覽（個案數、今日紀錄數、本週新增個案）
- 最近照護紀錄
- 釘選紀錄快速訪問
- 班級統計圖表

### 3.5 安全性設計

#### **Firestore Security Rules**
```javascript
// 範例：只有照顧者和管理員可以讀取個案資料
match /clients/{clientId} {
  allow read: if isCaregiverOrAdmin();
  allow read: if isClientFamily(clientId);  // 家屬只能看自己家人
  allow create, update, delete: if isCaregiverOrAdmin();
}
```

#### **權限分級**
- **管理員 (admin)**：完整系統權限
- **照顧者 (caregiver)**：可讀取所有個案、新增紀錄、查看班級資料
- **家屬 (family)**：僅能查看自己家人的資料
